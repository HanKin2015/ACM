1.打车问题
一辆的士最多载客4人，一支队伍最多4人，每支队伍必须在一辆车上，问怎么分配，最少需要多少辆的士？
思路：先将队伍的人数按照1、2、3、4进行分类，4人队伍肯定辆的士。2人的队伍两支可以坐一辆的士，坏的情况是多出一只2人队伍，它只能和1人的队伍拼车。1人队伍和3人队伍拼车，如果3人队伍多，就需要3人队伍的车(剩下的3人队伍每支一辆车)，如果1人队伍多，多出来的和多出来的2人队伍拼车。

2.同花顺
给定完整的一副牌，除去大小王，52张牌，输入多张牌，问是否存在同花顺。
思路：先将四种花色分类存储，可使用set自动排序方便查找连续五个连续的数字，注意A可以转换为1或14，然后进行判断，set有去重和排序功能，直接用两端值相减判断方便。

3.整数序列
给定一个长度为n(n<=16)的整数序列，整数绝对值夏普与等于100，问有多少长度大于等于2的序列是本质不同(位置不同不行，即不同位置的1和3组成只能算做1个答案)的上升子序列。
思路：时间不够，没有进行提交，思路应该没有错，鉴于n值取值很小，完全可以使用暴力求解，dfs深搜，列举所有的子序列，但最大的问题在于如何去重，后来我想明白了，可以使用map进行访问标记，即map<string, bool>，将子序列组成一个字符串，长度最大为1600，中间需要加间隔符，如_。

4. 字典序最大字符串
输入一个字符串只包含a-z小写字母，删除其中的字符，已有的字符至少保留一个，不会全部删除，使得剩下的字符字典序最大。
如abab,最后会是bab
如zzz，最后就是zzz
如abcab，最后是cab
